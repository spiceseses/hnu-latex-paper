\chapter{研究背景}
\vspace{7pt}

\section{问题概述}

随着云计算、分布式服务和在线业务系统的持续发展，服务器系统承担着数据处理、业务访问、服务调度和安全监控等核心任务。服务器运行状态一旦出现异常，可能导致服务响应延迟、业务中断、数据丢失或安全风险。Chandola 等在异常检测综述中将异常概括为与正常模式显著不一致的数据模式，为服务器运行状态识别提供了基本问题框架\cite{chandola2009anomaly}。He 等在自动化日志分析综述中指出，日志分析已经成为可靠性工程中支撑故障检测、问题定位和系统诊断的重要技术路径\cite{he2021survey}。因此，如何根据系统运行数据及时识别异常状态，是服务器运维与可靠性保障中的重要问题。

日志是服务器在运行过程中自动生成的事件记录，通常包含时间戳、节点信息、日志级别、组件来源和日志正文等内容。与 CPU 利用率、内存占用率等数值监控指标相比，日志能够更直接地反映程序执行路径、错误类型和异常上下文。Oliner 和 Stearley 对多类超级计算机系统日志的研究表明，系统日志中包含大量与故障、节点状态和运行环境相关的线索\cite{oliner2007supercomputers}。Xu 等围绕大规模控制台日志开展的问题检测研究进一步说明，日志事件的模板、频次和上下文关系可以用于识别系统异常状态\cite{xu2009detecting}。He 等在系统日志异常检测经验报告中也指出，日志分析能够为异常检测提供工程可用的数据基础\cite{he2016experience}。因此，基于日志分析的异常检测能够为故障预警、问题定位和运维决策提供依据。

然而，服务器日志并不是可以直接输入模型的规范化数据。首先，日志文本具有半结构化和非结构化特征，动态参数较多，同一类事件可能因 IP 地址、端口号、路径或编号不同而表现为多种文本形式。其次，日志按照真实时间产生，相邻事件之间的时间间隔并不均匀，异常爆发阶段可能在短时间内产生大量日志，而平稳运行阶段日志较为稀疏。再次，异常日志样本数量通常远少于正常样本，模型训练和评价容易受到类别不平衡影响。上述特点决定了服务器日志异常检测不能仅依赖简单关键词匹配或固定阈值规则，而需要结合日志解析、特征工程和时序建模方法进行系统化处理。

\section{研究意义}

本文围绕服务器日志异常检测问题，设计并实现一种基于日志分析和 Liquid 架构的异常检测系统。系统从原始日志数据出发，依次完成日志读取、数据清洗、模板解析、特征构建、模型训练、阈值校准和异常输出等步骤，形成较为完整的自动化检测流程。

本课题具有以下研究意义。第一，日志异常检测能够降低人工排查日志的工作量，提高运维人员发现异常和定位问题的效率。第二，将日志模板、事件频次、事件顺序和真实时间间隔统一纳入特征表示，有助于更全面地刻画服务器运行状态。第三，Liquid 架构具有连续时间建模特点，适合处理日志产生时间不均匀的问题。第四，基于验证集进行异常阈值校准，可以在一定程度上缓解正常样本与异常样本比例不均衡造成的误报或漏报问题。

\section{相关工作}

He 等将自动化日志分析任务概括为日志压缩、解析、异常检测、故障诊断和可靠性评估等方向，其中日志解析、特征表示和异常检测模型是本文工作最直接相关的三个方面\cite{he2021survey}。

在日志解析方面，常见方法通常先识别日志中的动态参数，再将语义相同但参数不同的日志归并为统一模板。Zhu 等在日志解析工具与基准研究中指出，解析结果质量会直接影响下游异常检测和故障诊断效果，因此需要在解析准确率、效率和适用场景之间进行权衡\cite{zhu2019tools}。Du 和 Li 提出的 Spell 采用最长公共子序列思想进行流式日志解析，适合处理持续到达的系统事件日志\cite{du2016spell}。He 等提出的 Drain 使用固定深度解析树提升在线解析效率，为大规模日志模板抽取提供了代表性树结构方法\cite{he2017drain}。这些研究说明，日志解析是将非结构化日志转化为可建模事件序列的关键步骤。

在特征表示方面，早期方法多采用事件计数、模板频次和统计分布描述系统状态。此类特征实现成本较低，但难以表达事件之间的顺序关系。Du 等提出的 DeepLog 将日志模板 ID 组织为事件序列，并利用 LSTM 学习正常执行路径的上下文模式\cite{du2017deeplog}。Meng 等提出的 LogAnomaly 同时利用序列信息和数量信息，说明日志异常既可能来自事件顺序变化，也可能来自事件统计分布变化\cite{meng2019loganomaly}。Guo 等提出的 LogBERT 将 BERT 的上下文表示引入日志异常检测，体现了语义表示方法在日志建模中的应用\cite{guo2021logbert}。对于服务器日志而言，仅使用频次特征容易忽略执行路径异常，仅使用序列特征又可能忽略局部事件密度变化，因此有必要构建融合多类信息的联合特征。

在异常检测模型方面，传统机器学习方法包括 PCA、Isolation Forest、One-Class SVM 等，适合处理统计特征或低维表示。Jolliffe 对主成分分析方法进行了系统阐述，PCA 常被用于从高维统计特征中提取主要变化方向\cite{jolliffe2002pca}。Liu 等提出的 Isolation Forest 通过随机划分隔离异常样本，为无监督异常检测提供了代表性方法\cite{liu2008isolation}。深度学习方法能够从事件序列中学习上下文模式，常见结构包括 LSTM、GRU、Transformer 及其变体。然而，这些模型通常把日志序列视为离散时间步序列，默认相邻事件之间的时间间隔相同，而真实服务器日志并不满足这一假设。Chen 等提出的 Neural ODE 为连续时间神经网络建模提供了基础形式\cite{chen2018neuralode}；Hasani 等提出的 Liquid Time-Constant Networks 进一步通过可变时间常数刻画连续动态过程\cite{hasani2021liquid}。本文引入 Liquid 架构，重点利用其连续时间动态建模能力，使模型能够在事件序列之外感知相邻日志之间的真实时间间隔。

综合来看，现有研究已经在日志模板抽取、日志序列建模和异常评分计算等方面形成了较成熟的技术路线，但在本科毕业设计场景下仍存在两个需要平衡的问题。一方面，系统需要具备较完整的工程流程，不能只停留在单一模型实验；另一方面，方法设计又需要保留可解释性，使日志模板、事件序列、时间间隔和异常评分之间具有清晰对应关系。基于此，本文没有单独追求复杂模型结构，而是围绕“结构化解析、联合特征、连续时间建模、动态阈值判定”构建完整系统。该定位使本文工作能够同时回应日志数据处理、模型训练和结果输出三个层面的需求。

\section{研究内容}

本文以服务器日志数据为研究对象，围绕日志异常检测系统的设计与实现开展研究，主要内容如下。

\subsection{日志预处理}

针对原始日志中字段不统一、冗余信息较多和动态参数复杂等问题，本文设计日志读取与预处理流程。该流程保留时间戳、节点、日志级别、组件、正文和标签等关键字段，完成缺失值处理、重复记录过滤、时间格式统一和文本规范化，为后续解析与建模提供稳定输入。

\subsection{日志解析}

针对非结构化日志正文，本文设计基于规则和正则表达式的日志解析模块。该模块识别 IP 地址、端口号、路径、数字编号、十六进制参数等动态内容，并将其替换为统一通配符，从而生成日志模板和事件 ID。通过模板化处理，系统能够降低文本稀疏性，并将日志流转换为可建模的事件序列。

\subsection{特征构建}

本文以时间窗口为样本单位，构建事件频次、事件顺序和时间间隔三类特征。事件频次用于刻画窗口内日志类型分布，事件顺序用于表达执行路径和上下文关系，时间间隔用于描述日志产生的真实节奏。三类特征共同构成模型输入，以支持对服务器状态变化的综合建模。

\subsection{模型训练}

本文采用 Liquid 架构构建异常检测模型。模型接收联合特征序列，输出窗口级异常评分。训练完成后，系统在验证集上遍历候选阈值，并选择使 F1 值较优的阈值作为最终判定边界，避免直接采用固定 0.5 阈值造成检测结果偏差。

\subsection{系统验证}

在方法研究基础上，本文实现端到端服务器日志异常检测系统，并在公开日志数据集上开展功能测试、对比实验和参数分析。实验内容包括系统模块正确性测试、训练过程分析、异常评分展示、混淆矩阵计算、模型对比以及输入序列长度和阈值对检测结果的影响分析。

\section{主要贡献}

本文的主要贡献包括以下四个方面。

\begin{enumerate}
    \item \textbf{构建了面向服务器日志异常检测的完整处理流程。} 本文将日志读取、清洗、解析、特征构建、模型训练、阈值校准和结果输出组织为统一系统，保证训练阶段和检测阶段的数据处理规则一致。
    
    \item \textbf{提出了融合频次、序列和时间间隔的联合特征表示。} 该表示兼顾局部日志分布、事件上下文和非规则时间信息，能够弥补单一特征表达不足的问题。
    
    \item \textbf{探索了 Liquid 架构在日志异常检测中的应用。} 本文利用 Liquid 架构对连续时间动态的表达能力，将真实时间间隔显式引入模型输入，使模型能够感知日志稀疏产生与异常爆发之间的差异。
    
    \item \textbf{设计了基于验证集的动态阈值校准策略。} 本文使用验证集评价指标确定异常判定阈值，使异常评分到二分类标签的转换过程具有数据依据。
\end{enumerate}

\section{论文结构}

本文共分为六章。

第一章为绪论，说明课题背景、研究意义、研究现状、主要研究内容和论文结构。第二章介绍服务器日志、日志解析、特征工程、异常检测评价指标和 Liquid 架构等理论基础。第三章进行系统需求分析与总体设计，明确功能需求、总体架构、核心模块和算法流程。第四章描述系统实现过程，包括日志预处理、模板解析、联合特征构建、Liquid 模型训练和结果输出。第五章开展系统测试与实验分析，验证系统功能完整性和模型检测效果。第六章总结全文工作，分析研究不足并提出后续改进方向。
